### 게임 오브젝트, 컴포넌트

- 씬
    - 게임의 장면이나 상태를 저장하는 단위. 하나의 거대한 게임을 씬 단위로 관리하며 코드를 이용해 이동이 가능함.
    
    **ex. title scene → loading scene → main scene** 
    
- 그렇다면 **오브젝트**란?
    - 정의
        - 이런 씬에 배치되는 하나의 물체를 지칭하는 단위!
        - 모든 게임 오브젝트는 위치/회전/크기를 제어하는 Transform 컴포넌트를 갖고 있음.
        - 게임 오브젝트는 이러한 컴포넌트를 묶어서 관리하고 접근할 수 있는 수단임.
    - 종류
        - 게임 오브젝트 메뉴
        - 빈 오브젝트
        - 3D, 2D
        - 이펙트
        - 오디오
        - 비디오
        - UI
    
    ⇒ 오브젝트에 컴포넌트라는 속성처럼 붙인다고 보면 됨.
    
    ⇒ 오브젝트는 기본적인 껍데기, 컴포넌트는 어떤 동작을 할지 정의하는 기능 블록
    
- 그렇다면 컴포넌트는 오브젝트에 어떻게 부여하는가?
    - C# 스크립트 파일 이용!
    - 즉, 게임 오브젝트에 컴포넌트를 부착하여 게임 오브젝트에 여러 기능을 부여
- Clear Flags
    - 오브젝트가 존재하지 않는 빈 배경을 어떻게 채울지 결정하는 요소
    - Depth only or Don’t Clear
        - 특정 색상이나 그림으로 Clear 해주지 않기 때문에 오브젝트가 이동했을 때 잔상이 남아있게 함.
    - Background
        - Clear Flags가 “Solid Color”일 때 배경화면의 색상을 나타내는 변수

유니티 이벤트 함수

- **Awake() → OnEnable() → Start()**
- Awake() # 일어나
    - 현재 씬에서 게임 오브젝트가 활성화 되어 있을 때 1회 호출 (컴포넌트가 비활성화 상태여도 게임 오브젝트가 활성화 되어 있으면 호출)
    - 데이터를 초기화 하는 용도로 사용
- Start()
    - 현재 씬에서 게임 오브젝트와 컴포넌트가 모두 활성화 되어 있을 때 1회 호출
    - 데이터를 초기화 하는 용도로 사용
- OnEnable()
    - 컴포넌트가 비활성화 되었다가 활성화 될 때 마다 1회 호출
- Update()
    - 현재 씬이 실행된 후 컴포넌트가 활성화되어 있을 때 매 프레임마다 호출
- FixedUpdate()
    - 프레임의 영향을 받지 않고 일정한 간격으로 호출
- OnDestroy()
    - 게임 오브젝트가 파괴될 때 1회 호출
    - 씬이 변경되거나 게임이 종료될 때도 호출됨.
